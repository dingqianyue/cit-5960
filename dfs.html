Depth-First Search (DFS)

Overview

Depth-First Search (DFS) is an algorithm for traversing or searching tree or graph data structures. It starts at the root node and explores as far as possible along each branch before backtracking.

DFS works for both directed and undirected graphs.

Algorithm Steps

  1. Start by pushing the root node into the stack.
  2. Pop a node from the stack to select the node for exploration. If the node is not marked as visited, mark it as visited and push all its adjacent nodes into the stack.
  3. Repeat step 2 until the stack is empty.

Applications

DFS is utilized in various situations such as path finding, scheduling problems, and cycle detection in graphs.

Types of Edges in DFS

Every DFS produces a Depth First Forest (a collection of trees) to that DFS.

  • Tree Edges: Lead to an undiscovered vertex and form the backbone of the DFS tree.
  • Forward Edges: Connect a vertex to a descendant in the DFS tree, other than via tree edges (not a part of the DFS tree).
  • Back Edges: Link a vertex back to one of its ancestors, important for detecting cycles in directed graphs. The existence of a back edge means that there is a cycle.
  • Cross Edges: Connect two vertices that are neither ancestors nor descendants of each other, usually linking nodes in different branches of the DFS tree. All other edges are cross edges.

Each edge can only be one of the types. The categories are mutually exclusive.
