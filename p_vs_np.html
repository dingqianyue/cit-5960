P vs NP Problems

Polynomial-time (P) Problems

Problems where the solution can be found in polynomial time are categorized as P. Examples include sorting, searching, and shortest path algorithms.

Nondeterministic Polynomial-time (NP) Problems

An example is prime factorization, which allows for the verification of a factor of an integer in polynomial time, but no known polynomial time algorithm exists for factorizing a given integer.

P is a subset of NP

The 0/1 Knapsack problem is classified as NP-Complete. It operates in pseudo-polynomial time O(nW), where its complexity becomes exponential when inputs are binary. The dynamic programming solution is linear in the value of W, but exponential in the length of W.

    • True. O(mn), where m is the length of string 1 and n is the length of string 2.

Complexity Classes

\(\mathcal{P}\) - class of problems solvable in polynomial time (\(O(\text{polynomial})\)) with respect to the input size.

Input Size Conventions

  • Array — length of the array
  • Graph — number of vertices and edges
  • String — number of characters
  • Single number — \(\log(\text{number})\)

\(\mathcal{NP}\), \(\mathcal{NP}\)-Hard, \(\mathcal{NP}\)-Complete

An \(O(nW)\) algorithm for the 0/1 knapsack problem does not categorize this problem as in \(\mathcal{P}\).

Decision Problems

A problem that yields a yes/no output.

Example: Hamiltonian Path Problem

Question: Given an undirected graph, does it contain a Hamiltonian path (a path that visits all vertices)?

Optimization Problems

Goals to maximize or minimize some function.

Examples include shortest path, edit distance, and the 0/1 knapsack problem.

Note: Solutions to decision problems can often be adapted to solve corresponding optimization problems.

Consider the decision version of the 0/1 knapsack problem:

  • Input: A knapsack with a given capacity \(W\)
  • Items each with a weight and a value
  • Question: Is there a selection of items such that the total value is at least \(V\) without exceeding the capacity \(W\)?

NP Definition

For decision problems: Given a "YES" instance and a "certificate" for this instance, the certificate can be verified in polynomial time.

Advice

Do not attempt to solve an \(\mathcal{NP}\) problem during an exam!

Hamiltonian Path Problem Example

Assuming a graph contains some Hamiltonian path, the certificate would be a sequence of vertices \(v_1, v_2, \ldots, v_n\).

  • Verification includes checking if all vertices are present and in order, with each consecutive pair forming an edge.
  • Using an adjacency matrix, checking if an edge exists between any two vertices is \(O(1)\).
  • Using an adjacency list, the entire verification takes \(O(n+m)\).

P vs NP

It is known that \(\mathcal{P}\) is a subset of \(\mathcal{NP}\). The unresolved question is whether \(\mathcal{NP}\) is a subset of \(\mathcal{P}\).

NP-Hard Problems

An NP-Hard problem is one such that if it can be solved in polynomial time, then every problem in \(\mathcal{NP}\) can be reduced to it and thus solved in polynomial time.

Reduction Example

Subset sum problem: Given a set \(S = \{a_1, a_2, \ldots, a_n\}\) and a target \(t\), determine if there is a subset whose sum equals \(t\).

Partition problem: Can \(S\) be divided into two subsets with equal sums?
